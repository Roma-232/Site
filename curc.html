Фундамент: Кто такой QA и зачем тестирование
2 часа

Теория

QA (Quality Assurance) — это процесс-ориентированный подход, направленный на предотвращение дефектов. QA-инженер работает над улучшением процессов разработки.

QC (Quality Control) — продукт-ориентированный подход, направленный на выявление дефектов в готовом продукте. QC-инженер (тестировщик) проверяет продукт.

Важно: Современный специалист часто совмещает обе роли. QA — это стратегия, QC — тактика.

Жизненные циклы:

  • SDLC (Software Development Life Cycle): Этапы разработки ПО (анализ, проектирование, разработка, тестирование, внедрение)
  • STLC (Software Testing Life Cycle): Этапы тестирования внутри SDLC

Ключевые термины

  • QA (Quality Assurance) — обеспечение качества
  • QC (Quality Control) — контроль качества
  • SDLC — жизненный цикл разработки ПО
  • STLC — жизненный цикл тестирования ПО
  • Тест-план — документ, описывающий стратегию тестирования

Виды тестирования: Выбираем правильный подход
2.5 часа

Теория

Классификация по доступу к коду:

  • Чёрный ящик: Тестируем без знания внутренней структуры (с точки зрения пользователя)
  • Белый ящик: Тестируем с полным доступом к коду (знаем внутреннюю логику)
  • Серый ящик: Частичное знание внутренней структуры

Уровни тестирования:

  1. Модульное: Тестирование отдельных компонентов (функций, классов)
  2. Интеграционное: Тестирование взаимодействия компонентов
  3. Системное: Тестирование системы как целого
  4. Приемочное: Тестирование заказчиком перед выпуском

2. Аудит безопасности исходного кода модуля перевода денег.

Тест-дизайн: Эквивалентное Разделение и BVA
3 часа

Теория

Тест-дизайн — этап создания тестовых случаев с использованием специальных техник.

Эквивалентное Разделение (Equivalence Partitioning):

Разделяем входные данные на классы эквивалентности. Все значения внутри одного класса считаются эквивалентными.

Анализ Граничных Значений (Boundary Value Analysis):

Тестируем значения на границах допустимых диапазонов. Ошибки часто скапливаются на границах.

Пример для поля "Возраст" [18, 65]:
Валидные значения:
18, 19, 64, 65
Невалидные значения:
17, 66

Практическое задание

Задача: Примените EP и BVA для поля "Количество товаров" [1, 99].

  1. Выделите классы эквивалентности
  2. Определите граничные значения
  3. Составьте таблицу тестовых значений
Решение:
Валидные классы: [1, 99]
Невалидные классы: (-∞, 0], [100, ∞)
Граничные значения: 0, 1, 2, 98, 99, 100
